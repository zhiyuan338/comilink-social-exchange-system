# ComiLink MVP 推荐开发顺序

## 总原则

优先跑通核心链路：

```text
触碰 NFC → 打开主页 → 自动收集 → 图鉴可见
```

不要先做复杂 UI、活动玩法或非必要后台功能。

---

## 开发顺序

### 1. 初始化项目

- Next.js
- TypeScript
- Tailwind CSS
- Prisma
- PostgreSQL

### 2. 建立数据库 Schema

- User
- Event
- SocialLink
- Collection

### 3. 实现登录系统

- QQ + 密码登录
- Session Cookie
- 登出
- 修改密码

### 4. 实现用户资料页

- 修改昵称
- 查看 NFC 链接
- 上传电子邮票
- 管理社交入口

### 5. 实现 NFC 访问页

- `/u/[token]`
- 展示目标用户资料
- 未登录可查看
- token 不存在显示 404

### 6. 实现自动图鉴收集

- 已登录访问他人主页后自动收集
- 自己访问自己不收集
- 重复访问不重复写入
- 无活动时仍可收集
- 保证并发安全

### 7. 实现图鉴页

- 双向可见
- 按活动筛选
- 展示收集时间

### 8. 实现后台用户管理

- CSV 导入用户
- 查看用户列表
- 重置密码为 QQ 号

### 9. 实现后台活动管理

- 创建活动
- 设置当前活动
- 保证只有一个当前活动

### 10. 移动端 UI 优化

重点页面：

- 登录页
- NFC 主页
- 图鉴页
- 用户资料页

### 11. 真机测试

测试内容：

- Android NFC 打开
- iPhone 兼容性
- 二维码备用访问
- 未登录访问
- 重复触碰
- 自己触碰自己
- 弱网访问

### 12. 部署上线

- 配置生产数据库
- 配置 HTTPS 域名
- 配置图片存储
- 写入 NFC 标签
- 进行现场测试
