# ComiLink MVP 验收标准

## 1. 用户与后台

- 管理员可以批量导入用户
- 导入用户时包含 QQ 号、昵称、初始密码
- 若导入时未指定密码，则初始密码默认为 QQ 号
- 用户可以使用 QQ 号和密码登录
- 用户可以修改密码
- 用户忘记密码后，管理员可以在后台将密码重置为该用户 QQ 号
- 用户首次登录后可以修改个人资料
- 用户可以上传电子邮票
- 用户可以配置多个社交入口
- 每个用户拥有唯一 NFC 链接

## 2. NFC 与扩列流程

- 主流 Android 手机可通过 NFC 打开用户主页
- 未登录用户仍可查看主页与电子邮票
- 已登录用户访问他人主页后可自动建立图鉴关系
- 用户访问自己的主页时不会建立图鉴关系
- 同一活动内不会重复收集
- 重复访问不会产生重复数据
- 无活动状态下系统仍可正常使用

## 3. 图鉴系统

- 图鉴关系为双向可见
- 双方都能在自己的图鉴页看到彼此
- 用户可以按活动查看已收集图鉴
- 用户修改资料后无需重新写 NFC 标签
- 用户修改电子邮票后，历史图鉴展示该用户当前最新邮票
- 任意用户之间在同一活动下仅允许存在一条图鉴关系

## 4. 活动系统

- 管理员可以创建测试活动
- 管理员可以创建正式活动
- 管理员可以将某个活动设置为当前活动
- 系统最多只有一个当前活动
- 新收集记录归属当前活动
- 切换当前活动不会影响历史图鉴记录
- 没有当前活动时仍允许收集，记录的 `eventId` 为空

## 5. 图片与文件

- 电子邮票支持 jpg / png / webp
- 非图片文件会被拒绝
- 图片超过限制时可以被服务端自动压缩
- 压缩后的图片可以在用户主页与图鉴页正常展示

## 6. 性能与稳定性

- NFC 页面首屏加载时间应尽量控制在 3 秒以内
- 图鉴建立操作应尽量在 1 秒内完成
- 图片资源能够稳定加载
- 500 人规模活动下系统可稳定完成主页访问与图鉴收集
- 数据库不会出现重复图鉴关系
- 并发访问下不会产生重复收集记录

## 7. 不属于 MVP 验收范围

以下能力不作为 MVP 验收目标：

- 即时聊天
- 动态社区
- 排行榜
- 活动任务
- 成就系统
- 用户注销账号
- 推送通知
- 多邮票系统
